Frequently Asked Questions

What is Ronin

  1. What language is Ronin written in?

    Ruby.

  2. What version of Ruby should I run Ronin on?

    We recommend running Ronin on Ruby 1.8.7-p302 or 1.9.2-p0

    Ronin can run on:

    • Ruby: 1.8.7, 1.9.1, 1.9.2
    • JRuby: 1.5.1
    • Rubinius: edge

    However, Ronin does not support Ruby 1.8.6, due it's slowness and incompatibilities. If you are still using 1.8.6, we recommend you upgrade to either Ruby 1.8.7 or 1.9.2.

  3. What is the easiest way to install Ruby?

    Ruby Version Manager (RVM)

  4. What are some good resources for learning Ruby?

    LearnToProgram is an excellent book and tutorial which provides a solid introduction to Ruby.

    There's also the free and more technical Pragmatic Programmer's Guide to Ruby, which many Rubyists first learned from.

    If you're more of a hands-on learner, there is also the Try Ruby in your Browser (Beta) interactive tutorial. If your still wanting more hands-on challenges, try Ruby Koans.

  5. Is Ronin a Framework?

    Not exactly. Ronin can be thought of more as a Platform for other people's code, exploits, payloads or research. Ronin provides the convenience methods, the additional libraries and a means to distribute the code, exploits and payloads.

    One of Ronin's design goals was to create a Platform that facilitated and encouraged third-party work.

  6. What is Ronin licensed under?

    GPL-2

  7. How well documented is Ronin?

    Ronin has extensive YARD documentation covering it's API. Additional guides and screencasts are always in the works.

  8. How readable is Ronin's code-base?

    Other projects have been criticized for having poor programming conventions or a lack of documentation. To avoid this, Ronin enforces a set of coding style guidelines to ensure that all source-code is clean, readable and maintainable.

  9. Where is Ronin's code-base hosted?

    Ronin and it's many libraries are hosted on GitHub. For more information on using GitHub and a complete listing of Ronin's projects, see the Contribute section.

  10. Does Ronin have a test-suite so that I can ensure that Ronin is bug free?

    Yes, Ronin comes with extensive RSpec specification tests.

  11. Does Ronin have a command-line interface (CLI)?

    Why yes. Ronin's design allows for the libraries of Ronin to add sub-commands to the main ronin command. To view all available commands simply run the following command: 

    $ ronin help

    To view the help information of a specific command:

    $ ronin help command_here

Exploitation

  1. Does Ronin have prepackaged exploits?

    Ronin does not ship with any builtin Exploits. Although, Ronin makes it rather easy to write your own exploits using ronin-exploits and distribute them in Overlays.

    It takes a lot of time and energy to write and weaponize exploits for known vulnerabilities; let alone find new vulnerabilities. So, we decided to leave the exploit writing up to the skilled security researchers of the world, and let them decide how and when to distribute their own exploits.

  2. Does Ronin provide an autopwn-like feature?

    Ronin does not provide an autopwn-like feature. Features such as autopwn make assumptions about the workflow of a "real attacker", and attempt to automate it. Naturally, the "real attackers" will change their tactics and use new types of information gathering (ex: Datamining Social Networks) and attacks (ex: Phishing) which may not cleanly fit into existing Exploitation Frameworks; thus rendering autopwn-like features outdated and ineffective.

    Ronin was designed to not impose a workflow onto the user. Instead, Ronin provides useful APIs, commands and a Database for the user to leverage however they want to.

Platform and Overlays

  1. What is this Ronin Platform?

    The Platform simply manages Overlays and their contents, allowing users to install, update and remove Overlays.

  2. What exactly are Overlays?

    Overlays are directories which can contain Objects (such as Exploits or Payloads) or Extensions (small groupings of code) that can be used within Ronin. An Overlay also contains a small XML file named ronin.xml that describes the Overlay, it's authors and it's licensing.

  3. How are Overlays hosted?

    An Overlay can be hosed on any Subversion (SVN), Mercurial (Hg), Git or Rsync server.

    One could easily host an Overlay on free source-code hosting sites such as, GitHub, Gitorious, Google Code or SourceForge.

  4. How can I install an Overlay?

    Simply pass the URL of the Overlay to the ronin install sub-command. 

    $ ronin install svn://www.example.com/overlay

    If you have previously checked out an Overlay, then simply pass the path of the local Overlay to the ronin add sub-command. 

    $ ronin add path/to/local/overlay
  5. How can I generate a skeleton Overlay?

    You can generate a skeleton Overlay by simply running the following command provided by the Ronin Gen library: 

    $ ronin-gen overlay path/to/overlay/name

    Or if you wish to specify additional Overlay metadata: 

    $ ronin-gen overlay -n Overlay-Name -L GPL-2 \
  -m "Author " \
  -S http://svn.mysite.com/overlay \
  -U http://myblog.com \
  -D "This is my Overlay, enjoy." \
  path/to/overlay
